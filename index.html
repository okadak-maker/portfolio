自己紹介

福祉の現場って、まだまだ紙とExcelが多いんです。「この作業、毎週やってるけど自動化できないかな」という素朴な疑問から、LINE WORKS Bot、社内ポータル、AIチャットボットと、必要なものを片っ端から作ってきました。

特別な技術力があるわけじゃないです。ただ、現場の人が本当に困っていることを聞いて、それを解決するために必要な技術を覚えて、動くものを作る。その繰り返しです。

最近はAIエージェントの活用にハマっていて、LLMに長期記憶を持たせるシステムを自作したり、AIコーディング環境を徹底的にカスタマイズしたりしています。

Ghost Memory Engine

運用中

「AIに記憶を持たせたかった」

AIアシスタントと毎日やりとりしているうちに、「昨日話したこと覚えてないのか…」と思うことが増えた。だったら自分で記憶システムを作ろうと、脳科学の論文を読み漁って、情動・忘却曲線・連想ネットワークをコードに落とし込んだ。今ではMCPサーバーとして複数のAI環境に組み込んで使っている。

  • ベクトル検索 + 情動・鮮度・連想によるランキング
  • 夢生成（バロウズ式カットアップ）で記憶を再構成
  • 300件以上の記憶を蓄積、日常的に稼働中
PythonSQLitesentence-transformersFastMCP

KarasuBot

運用中

「外出中でもAIにコードを書かせたい」

AIコーディングアシスタント（Antigravity）をスマホから遠隔操作したかった。DiscordのBotとしてChrome DevTools Protocolでブラウザを制御し、スクショ取得・自動承認・Git操作など26の機能を実装。VSCode拡張の双方向ブリッジも自作した。

  • Chrome DevTools Protocol で実ブラウザを制御
  • 常駐プロセスとして24時間稼働

LINE WORKS Bot 基盤

運用中

「Cronジョブの結果、誰も見てないよね」

バッチ処理の結果がログに埋もれて誰も見ない問題を、LINE WORKSに自動転送することで解決。Windows Updateのパッチ情報をMSRC APIから取得して毎月自動通知する仕組みも組み込んだ。

  • Cron → LINE WORKS 転送ブリッジ
  • MSRC API連携でWUパッチ管理を自動化
  • JWT認証、マルチBot構成
PythonFastAPILINE WORKS APIDockerCloud Run
